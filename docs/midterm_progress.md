# 中期考核阶段性进展素材

## 阶段目标

论文题目为《基于语义联想机制的动态知识图谱记忆系统方法与实现》。开题后，当前阶段围绕“动态知识图谱记忆”和“语义联想检索”两个核心问题，先实现了一个可运行的最小原型，用于验证技术路线的可行性。

## 已完成工作

1. 完成项目工程初始化。
   - 已建立独立代码仓库 `SAM`。
   - 核心代码采用 `src/sam/` 包结构组织，避免所有代码堆叠在单一目录。
   - 已补充中文 README、运行脚本、测试文件和实验报告目录。

2. 完成记忆系统基础数据结构。
   - 设计了记忆节点 `MemoryNode`，包含文本、摘要、关键词、标签、来源、时间戳、使用次数、置信度和 embedding。
   - 设计了语义边 `MemoryEdge`，包含起点、终点、关系类型、边权和建边原因。
   - 使用 SQLite 实现本地持久化，能够保存节点、边和检索日志。

3. 完成按需动态建图机制。
   - 系统不会在写入阶段构建完整图谱，而是在记忆节点被检索激活后，再围绕种子节点补充语义边。
   - 当前支持共享实体、关键词重叠、embedding 相似度三类建边依据。
   - 该策略回应了开题答辩中专家提出的“建图时间成本可能较高”的问题。

4. 完成两阶段语义联想检索。
   - 第一阶段使用向量相似度检索种子记忆节点。
   - 第二阶段沿语义边扩展相关记忆，形成联想子图。
   - 检索结果包含排序分数、相似度分数、图扩展分数、使用频率分数和路径解释。

5. 完成初步公开基准风格实验。
   - 实验样本字段设计对齐 MultiHop-RAG、HotpotQA 和 MuSiQue 等公开多跳问答数据集。
   - 已实现纯向量检索和联想图检索对比。
   - 已生成实验报告 `reports/experiment_results.md`。

## 初步实验结果

当前小样本实验包含 3 个跨文档/多跳问答查询，覆盖 MultiHop-RAG、HotpotQA、MuSiQue 三类公开基准的任务形态。结果如下：

| 指标 | 纯向量检索 | 联想图检索 |
| --- | ---: | ---: |
| 支持证据召回率 | 0.667 | 1.000 |
| 命中支持证据数 | 4 | 6 |

联想图检索相比纯向量检索新增命中 2 条有效支持证据，说明在部分问题中，第二跳证据与原始问题并不直接相似，但与第一跳证据存在共享实体或语义关联。通过图扩展可以补回纯向量检索漏掉的证据链。

## 阶段性结论

当前结果表明，开题报告中提出的“向量召回 + 语义关联路径扩展”的路线具备初步可行性。纯向量检索适合作为高效入口，但在跨文档多跳推理中容易遗漏间接相关信息；引入按需构建的语义边后，系统可以沿被激活记忆进行联想扩展，从而提升支持证据召回。

目前系统仍是最小原型，实验规模较小，尚不能作为最终结论。但它已经形成了可运行代码、可解释检索路径和初步对比结果，能够支撑后续继续扩展到真实公开数据集和更完整的论文实验。

## 后续安排

1. 接入真实 MultiHop-RAG、HotpotQA、MuSiQue 数据集子集，扩大实验规模。
2. 优化记忆节点抽取，从完整文档中自动提取摘要、关键词、实体和关系。
3. 增强图谱更新策略，引入边权衰减、噪声过滤和使用频率反馈。
4. 实现多智能体共享记忆接口，为 Reading Agent、Summary Agent 等角色提供统一读写能力。
5. 设计类比推理触发器，用于检索结构相似的历史案例和推理路径。
